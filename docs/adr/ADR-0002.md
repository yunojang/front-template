# ADR-0002 – UI 접근성: Radix UI + TailwindCSS

- 상태: Accepted
- 날짜: 2025-01-20

## 문제

프로젝트는 다양한 인터랙션(모달, 탭, 토글, 리스트 등)을 포함하며, 초보 팀이 직접 접근성(A11y)을 구현하기 어렵다. 또한 TailwindCSS를 활용해 빠르게 스타일링할 수 있어야 한다.

## 의사결정

- **Radix UI**를 기본 상호작용 컴포넌트(Modal, Tabs, Select, Toast 등)에 사용한다.
  - WAI-ARIA 스펙을 준수하며 키보드 탐색과 포커스 관리를 내장
  - Headless 컴포넌트라 Tailwind와 쉽게 조합 가능
- **TailwindCSS**로 디자인 토큰과 레이아웃을 정의한다.
  - 공통 유틸리티(`bg-surface-*`, `text-muted` 등)를 커스텀 레이어에 선언
  - 초보 팀이 클래스 조합만으로 UI를 빠르게 확장할 수 있음
- 공통 UI 래퍼(`shared/ui/Button`, `Dialog`, `Tabs` 등)를 만들어 일관된 스타일과 접근성 속성을 제공한다.

## 고려했던 대안

1. MUI / Chakra UI 등 완성형 컴포넌트 라이브러리
   - Pros: 디자인 토큰과 컴포넌트가 완비
   - Cons: 커스터마이징 비용이 크고 Tailwind와 혼합 시 중복 스타일 우려
2. 완전 커스텀 구현
   - Pros: 자유도 높음
   - Cons: 접근성 구현 난이도와 유지보수 부담이 큼

## 결과

- Radix 기반 Dialog/Select/Tabs/Toast 래퍼를 프로젝트에 포함
- Tailwind 테마 변수를 정의하여 공통 색상/타이포를 재사용
- Storybook을 통해 컴포넌트 카탈로그를 확장할 수 있도록 `pnpm story` 스크립트 제공

## 추적 항목

- 라이트/다크 테마 스위치를 도입할 지 추후 검토
- Radix 컴포넌트 추가 도입 (DropdownMenu, Tooltip 등)을 Storybook에서 관리
